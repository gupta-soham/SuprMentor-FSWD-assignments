Folder Architect

Assignment — 19th March 2026

Goal

Plan a clear folder layout for a full-stack web app before writing code. Good structure keeps features easy to find, scales with team size, and matches how build tools expect files to be organized.

Example: Vite + React (client) + Express API (server)

What each top-level area is for

  • client/ — Everything the browser runs (UI, styles, client-side routing).
  • server/ — HTTP API, auth, database access; no React components here.
  • shared/ (optional) — Types, validation schemas, constants used by both sides.
  • docs/ (optional) — Architecture notes, API contracts, onboarding.

Rules of thumb

Co-locate by feature when the app grows
  • Instead of one giant components/, use features/cart/ with its own components + hooks.
Keep environment secrets out of Git
  • Commit .env.example with dummy keys; real .env stays local.
One responsibility per folder
  • routes/ only wire URLs to handlers; heavy logic lives in services or modules.
