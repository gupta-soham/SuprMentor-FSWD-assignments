1. Client-Server Architecture

The client-server model is the foundational architecture of the web. The client (typically a browser) sends requests to a remote server, which processes them and returns the appropriate response.

Components explained:

  • Client (Browser): Initiates requests, renders HTML/CSS, executes JavaScript.
  • DNS Server: Translates human-readable domain names (e.g. google.com) into IP addresses.
  • Load Balancer: Distributes incoming traffic across multiple servers for reliability.
  • Web Server: Handles HTTP connections, serves static files, and proxies to the application server.
  • Application Server: Runs business logic, processes data, calls APIs.
  • Database: Stores and retrieves persistent application data.

2. What Happens When You Open a Website

Below is the step-by-step sequence that occurs from the moment you type a URL (e.g. https://www.example.com) into the browser until the page is displayed.

3. Step-by-Step Explanation

  1. URL Entry: The user types a URL (e.g. https://www.example.com) into the address bar and presses Enter.
  2. URL Parsing: The browser extracts the protocol (https), domain (www.example.com), and path (/).
  3. DNS Resolution: The browser queries a DNS server to translate the domain name into a numeric IP address.
  4. IP Response: The DNS server responds with the IP address of the target server.
  5. TCP Handshake: A three-way handshake (SYN → SYN-ACK → ACK) establishes a reliable connection between the browser and the server.
  6. TLS/SSL Handshake: For HTTPS, an additional handshake negotiates encryption keys to secure the connection.
  7. HTTP Request: The browser sends an HTTP GET request to the server, including headers like Host, User-Agent, and Accept.
  8. Server Processing: The web server receives the request and forwards it to the application server for business logic.
  9. Database Query: If dynamic data is needed, the application server queries the database.
  10. Data Response: The database returns the requested data to the application server.
  11. Response Generation: The application server generates the HTML response and sends it back through the web server.
  12. HTTP Response: The server sends an HTTP response (status code 200 OK) containing the HTML document.
  13. HTML Parsing: The browser parses the HTML and identifies additional resources (CSS, JS, images) that need to be fetched.
  14. Resource Fetching: The browser sends additional HTTP requests for stylesheets, scripts, and images.
  15. Resources Received: The server returns all the requested static assets.
  16. Rendering: The browser constructs the DOM, applies CSS styles, and executes JavaScript to render the page.
  17. Page Displayed: The fully rendered webpage is displayed to the user.

  • HTTP/HTTPS: Application-layer protocol for transferring web content. HTTPS adds encryption via TLS.
  • TCP: Transport-layer protocol ensuring reliable, ordered data delivery through connections.
  • IP: Network-layer protocol responsible for addressing and routing packets across networks.
  • DNS: Translates human-readable domain names into machine-readable IP addresses.
  • TLS/SSL: Provides encryption, authentication, and data integrity for secure communications.
